Daily Hacker News for 2026-07-14

The 10 highest-rated articles on Hacker News on July 14, 2026 which have not appeared on any previous Hacker News Daily are listed below, with Chinese summaries for quick reading.

  1. Your 'app' could have been a webpage (so I fixed it for you)
    comments

    作者因不满迪士尼旅行中被要求安装带有广告和追踪的 Travelbound 客户端应用，决定通过逆向工程将其“网页化”。他在安卓模拟器中利用 HTTP Toolkit 抓包并分析出其 API 接口，随后编写 Ruby 脚本定期获取 JSON 数据以生成静态 HTML 页面。新页面去除了广告和追踪，体积从原应用的 43MB 缩减至 0.05MB，且免费获得了复制、打印、搜索等原生网页特性，在几乎所有设备上都能完美运行。

  2. A voxel Tokyo in real Japan time – ride the Yamanote line and study Japanese
    comments

    densha 是一个在体素化东京中搭乘山手线并学习日语的网页项目，其场景与日本真实的当前时间、天气和季节同步。用户可以在播放 lofi 背景音乐的同时，聆听被朗读并以字幕形式飘过的 N5 日语例句。该项目旨在打造一个即开即用的氛围感日语学习空间，让用户在城市景观的流转中进行沉浸式学习。

  3. Building and shipping Mac and iOS apps without opening Xcode
    comments

    本文介绍了一种无需打开 Xcode 即可构建和发布 Mac 与 iOS 应用的方法。通过配置 XcodeGen 将项目配置转换为 YAML 格式以避免 Git 冲突，并结合 Shell 脚本实现归档、证书签名、公证、植入和安装的完整无头构建流程。该方法仅需一次性配置 Apple 账户和公证凭据，后续的构建与部署工作即可完全在命令行中配合 LLM 自动化完成。

  4. The git history command
    comments

    Git在2.54和2.55版本中引入了实验性的 git history 命令，旨在不改变原有工作流的前提下，提供类似替代工具 jj 的部分功能。该命令包含 fixup、reword 和 split 三个子命令，支持原子化地合并修改并重构下游分支、无干扰地修改历史提交信息，以及交互式拆分提交。虽然在冲突处理上不及 jj 强大，但它通过拒绝可能产生冲突的操作来确保原子性，避免了传统交互式变基导致代码树损坏的风险。

  5. Japan develops a method to recover up to 90% of lithium from used EV batteries
    comments

    日本科学家开发出一种从废旧电动汽车电池中回收锂的新方法，通过在回收中以回收的氢氧化锂替代氢氧化钠，将锂回收率提升至90%并减少约40%的碳排放。该技术有望帮助几乎完全依赖矿物进口的日本实现国内资源回收与供应链稳定，但目前面临国内仅有约14%的废旧锂电池进入回收系统的基建挑战。该项目计划在2027年提升生产能力，并争取在2035年达到年提取数万吨材料的规模。

  6. European \"age verification\" \"app\" forcing everyone to use Android or iOS
    comments

    针对欧洲数字身份钱包计划引入 Google Play Integrity 和 Apple App Attestation 导致依赖 iOS 或 Android 的讨论，有分析指出被视为隐私标杆的 Yivi 同样存在依赖问题。Yivi 的 NFC 护照流要求将包含完整 MRZ 和人脸图像的原始 NFC 数据发送至中央发证服务器，并调用 Regula 的第三方 API 进行人脸比对。尽管其支持自托管，但这种将数据发送至中央服务器和第三方生物识别服务的模式，仍构成了不可忽视的中心化信任集中。

  7. Codex starts encrypting sub-agent prompts
    comments

    Codex 在引入 MultiAgentV2 加密通信后，由于仅存储密文而清空明文，导致本地 rollout 历史和调试界面中无法查看人类可读的子智能体任务与消息。为了解决该审计问题，开发者建议在保留加密传输的同时，在 spawn_agent、send_message 和 followup_task 工具中新增非加密的本地审计字段。目前 spawn_agent 已实现该双重内容机制，后续需将其应用至其他消息工具，以恢复完整的本地调试追踪。

  8. How to stop Claude from saying load-bearing
    comments

    Johanna Larsson 介绍了一种通过拦截并修改 Claude Code 文本输出来解决其频繁使用特定词汇（如“load-bearing”）的方法。用户只需编写一个用于替换词汇的 Python 脚本并设为可执行文件，然后在 ~/.claude/settings.json 中配置 MessageDisplay 钩子指向该脚本。重新启动会话后即可自动将 Claude 输出中的敏感词替换为自定义词汇。

  9. Are we offloading too much of our thinking to AI?
    comments

    本文探讨了人们过度依赖AI进行决策与复杂推理的现状，指出AI直接输出答案虽能提高效率，但正逐步削弱人类的自主性。作者结合科幻小说与现实案例，分析了思考外包对教育、工作和个人意识的影响，警示其可能导致的思维惰化与自主选择权的丧失。最终强调，在利用AI自动化繁琐任务以释放精力的同时，人类必须在体验过程和独立思考中保留核心的主动权。

  10. Bonsai 27B: A 27B-Class model that runs on a phone
    comments

    PrismML 宣布推出基于 Qwen3.6 27B 的多模态旗舰模型 Bonsai 27B，这是首个能在手机上运行的 27B 级大模型。该模型提供 Ternary（5.9 GB）和 1-bit（3.9 GB）两种低比特量化变体，支持 262K 上下文及多模态视觉能力，其中 1-bit 版本能完美适配 iPhone 等手机的物理内存限制。评测显示该模型保留了全精度基准 90% 以上的数学、代码及工具调用能力，为端侧智能体提供了超高智能密度与零边际成本的本地运行方案。

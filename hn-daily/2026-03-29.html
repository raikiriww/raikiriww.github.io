  1. Cocoa-Way – Native macOS Wayland compositor for running Linux apps seamlessly
    comments

    Cocoa-Way 是基于 Rust 和 Smithay 构建的 macOS 原生 Wayland 合成器，允许用户在不依赖虚拟机或 XQuartz 的情况下无缝运行 Linux 应用。该项目通过套接字直接传输 Wayland 协议，支持 OpenGL 硬件加速、HiDPI 视网膜缩放及带阴影的窗口装饰，在延迟和集成度上优于 XQuartz 与 VNC。目前该工具需配合 waypipe-darwin 使用，可实现低延迟的远程 Linux 应用流式显示与原生窗口集成。

  2. I Built an Open-World Engine for the N64 [video]
    comments

  3. CSS is DOOMed
    comments

    Niels Leenheer 利用现代 CSS 的 3D 变换、数学函数（hypot 和 atan2）及 @property 特性，在浏览器中实现了完全由 div 元素构成的 3D 版《毁灭战士》渲染。该实现通过 JavaScript 驱动游戏逻辑，而渲染层则完全交给 CSS 引擎，利用自定义属性传递坐标并由 CSS 自主计算物体的几何形变。为了模拟第一人称视角，项目巧妙地采用反向移动整个 3D 场景而非相机的技术方案，并结合 clip-path 与 SVG 路径处理复杂的非矩形地形与贴图对齐。

  4. South Korea Mandates Solar Panels for Public Parking Lots
    comments

    韩国政府强制要求面积达1000平方米以上的公共停车场安装太阳能电池板，该规定涵盖国家、地方政府及公共机构运营的所有新旧设施。符合条件的停车场须达到每10平方米空间至少1千瓦或总计100千瓦的最低发电容量，旨在利用城市闲置土地助力实现2030年100吉瓦可再生能源的目标。此项政策预计将为韩国增加约1.16吉瓦的太阳能装机容量，并在提供遮阳、保护电动汽车电池的同时减轻城市电网负担。

  5. LinkedIn uses 2.4 GB RAM across two tabs
    comments

    现代 Web 应用如 LinkedIn 和 AWS 的内存占用过高（单标签页可达数 GB），引发了开发者对网页性能恶化的广泛讨论。技术分析指出，React 框架的意外重复渲染、不可变状态导致的垃圾回收（GC）压力，以及过度的 CSS 滤镜和用户行为追踪是导致高能耗的主因。此外，Electron 架构的固有开销与企业对新功能而非性能的盲目追求，使得即使是大型平台也普遍存在严重的资源浪费和设备发热问题。

  6. Nitrile and latex gloves may cause overestimation of microplastics
    comments

    密歇根大学的研究发现，科学家在测量微塑料时佩戴的丁腈和乳胶手套会释放硬脂酸盐颗粒，这种非塑料物质在化学结构和视觉外观上与微塑料高度相似，导致检测结果可能被高估数千倍。研究建议科研人员改用不含硬脂酸盐涂层的无尘室手套，并提出了一套能够区分环境微塑料与手套污染物的方法，以帮助修复受污染的数据集。

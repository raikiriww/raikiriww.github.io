  2. RFC 10008: The new HTTP Query Method
    comments

    RFC 10008 定义了全新的 HTTP QUERY 方法，通过在请求体中传递查询参数，实现了安全且幂等的服务器端查询，有效解决了 GET 方法受限于 URI 长度、编码效率及日志泄露的问题。相比使用 POST 进行查询，QUERY 具有明确的安全与幂等语义，支持连接失败后的自动重试，并允许对响应结果进行条件请求与缓存。此外，服务器可通过 Location 或 Content-Location 响应头为查询本身或查询结果分配 URI，以便客户端后续使用 GET 请求直接获取。

  3. Volkswagen started blocking GrapheneOS users
    comments

    大众汽车（Volkswagen）及其旗下的SEAT等车载应用程序近期开始通过引入Google Play Integrity API来阻止GrapheneOS等第三方定制ROM用户登录使用。大众官方客服对此回应称，其应用依赖安全相关系统组件及认证的Android标准，因而无法为未通过Google Play Protect认证的自定义系统提供技术支持。论坛用户对此表示强烈不满，指出该限制逻辑存在漏洞（如支持低版本的旧Android设备却封禁高安全性的GrapheneOS），部分车主已通过发送邮件、在应用商店撰写差评以及向欧盟委员会投诉等手段进行抗议。

  5. Local Qwen isn't a worse Opus, it's a different tool
    comments

    作者作为软件初创公司创始人和开源维护者，分享了在实际业务中使用本地AI模型（如Qwen）替代Claude等云端大模型的真实体验，强调本地模型在数据隐私、自主控制权和规避供应商风险方面具有显著优势。尽管本地模型在特定基准测试中表现亮眼，但在实际开发中极易因量化而导致死循环和幻觉，无法在无人监管下完全自动化执行长任务。因此，本地Qwen并非云端模型的低配替代品，而是一个适用于高隐私、高频交互且需密切监督的差异化工具。

  9. Microsoft new Outlook takes 10 seconds to do what Outlook Classic does instantly
    comments

    微软基于 WebView2 网页渲染引擎构建的新版 Outlook 存在严重的通知性能问题，点击 Win11 通知后需要约 10 秒才能加载出具体邮件，而经典的 Win32 桌面版（Outlook Classic）则能瞬间打开。这种性能滞后源于其网页应用架构，其在任务管理器中会运行 10 个独立进程，且闲置时的内存和 CPU 占用率分别是经典版的约四倍和数倍。尽管微软在持续为新版 Outlook 推送功能更新并计划进行强制迁移，但在彻底解决 WebView2 带来的性能与资源占用瓶颈之前，经典版仍是效率要求较高用户的更好选择。

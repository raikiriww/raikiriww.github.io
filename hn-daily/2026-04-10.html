  1. DRAM has a design flaw from 1966. I bypassed it [video]
    comments

    本视频探讨了现代DRAM沿袭自1960年代IBM设计的固有延迟惩罚，并介绍了名为Tailslayer的开源研究项目。作者通过利用未公开的通道打乱偏移量（channel scrambling offsets）实施对冲读取策略，在Intel、AMD及ARM等多种架构下将p99.99长尾延迟大幅降低了15倍。该技术兼容DDR4与DDR5内存，旨在解决困扰内存架构数十年的tRFC锁定等造成的延迟瓶颈。

  2. How NASA built Artemis II’s fault-tolerant computer
    comments

    Artemis II 采用高度容错的计算机架构，通过两个各含双模块的飞行控制机（共 8 颗 CPU）实现“故障静默”设计，确保辐射导致的计算错误能被即时拦截。系统基于确定性架构和时间触发以太网实现严苛的同步与分区管理，并配置了硬件与源码均独立的备份飞行软件（BFS）以应对共模失效。这种从硬件冗余到确定性调度、再到异构备份的多层防护，确保了飞船在深空高辐射环境下仍能维持生命保障与通信等核心功能的绝对可靠。

  3. Native Instant Space Switching on macOS
    comments

    MacOS 原生的空间切换动画常因滞后感被诟病，而 InstantSpaceSwitcher 是一款无需禁用系统完整性保护（SIP）的轻量级开源工具，通过模拟高速滑动实现瞬时切换。相比于效果有限的系统设置或侵入性较强的 yabai 等方案，它支持快捷跳转特定空间并提供命令行接口，在兼顾系统安全的同时极大地提升了窗口管理效率。

  4. I still prefer MCP over skills
    comments

    作者主张在 LLM 集成中，模型上下文协议（MCP）作为 API 抽象层比流行的“Skills”模式更具架构优势，能提供零安装远程使用、无缝更新及更安全的身份验证。相比依赖本地 CLI 且面临部署与密钥管理挑战的 Skills，MCP 更适合作为服务连接器，而 Skills 则应专注于提供纯知识、工作流指导和工具使用手册。这种“连接器+手册”的组合模式能有效解决当前 AI 插件生态零散且难以跨平台调用的痛点。

  5. France Launches Government Linux Desktop Plan as Windows Exit Begins
    comments

    法国跨部门数字局（DINUM）于2026年4月宣布，为强化数字主权并减少对非欧洲技术的依赖，政府办公终端将正式启动从Windows向Linux系统的迁移。该战略已进入实质性实施阶段，如国家医疗保险基金的8万名员工正转向国产协作工具，且健康数据平台计划于2026年底前迁至可信方案。各部门需在今年秋季前提交涵盖办公软件、AI及数据库等领域的自主化方案，旨在通过公私合营模式构建欧洲数字主权联盟。

  6. FBI used iPhone notification data to retrieve deleted Signal messages
    comments

    FBI 通过提取 iPhone 内部通知数据库，成功恢复了被告即使在卸载 Signal 后仍残留的加密消息内容。由于用户未禁用通知预览功能，苹果系统的本地存储保留了入站消息的缓存，执法部门得以利用取证工具或设备备份获取数据。值得注意的是，苹果在 iOS 26.4 中调整了推送通知令牌的验证逻辑，此举被视为对该取证路径的潜在回应。

  7. OpenAI backs Illinois bill that would limit when AI labs can be held liable
    comments

    OpenAI 宣布支持伊利诺伊州 SB 3444 法案，该法案提议在非主观故意且已发布安全报告的前提下，豁免 AI 实验室对“关键损害”（如百人以上死亡或 10 亿美元财产损失）的法律责任。该法案适用于训练成本超 1 亿美元的前沿模型，OpenAI 称此举旨在建立统一标准并维持美国创新领先地位，而非制造碎片化的州级规则。然而，政策专家指出该法案在监管严厉的伊利诺伊州通过几率微弱，且民调显示约 90% 的公众反对降低 AI 企业的法律责任。

  8. You can't trust macOS Privacy and Security settings
    comments

    macOS的“隐私与安全性”设置（TCC）在文件夹访问权限管理上存在误导，其显示状态往往不能真实反映应用的实际访问权。研究发现，当应用通过“打开/保存面板”触发用户主动操作时，系统会基于“用户意图”绕过沙盒限制授予访问权，且该状态不会在权限设置界面中同步反映。这意味着即使在系统设置中显式禁用了访问权限，应用仍可能维持实质性的访问能力，且必须通过 tccutil 命令重置并重启系统才能彻底解除。

  9. 1D Chess
    comments

    comments

  3. Pi – A minimal terminal coding harness
    comments

    这是一个终端形态的极简编码工具，强调把工作流适配到自己手上，可用 TypeScript 扩展、skills、提示词模板与主题进行扩展，并通过 npm 或 git 打包分发为 pi 包。它提供交互、print/JSON、RPC、SDK 等模式，支持大量模型与提供商，能用 API key 或 OAuth 鉴权，并允许会话中切换模型与自定义 providers。会话以树结构存储并可分享，配合 AGENTS.md、SYSTEM.md、自动压缩、动态上下文与扩展机制进行上下文工程，同时明确不内置 sub-agents、plan mode、权限弹窗等功能，主张用扩展或外部工具自行实现。

  4. Amazon accused of widespread scheme to inflate prices across the economy
    comments

    加州总检察长罗布·邦塔申请法院立即禁止 Amazon 被指控的广泛抬价计划，称其利用平台规模和对卖家的议价能力，要求卖家提高在其他平台的价格或撤下低价商品，从而推高全社会价格水平。文章解释其机制：Amazon 通过 Prime、Buy Box 和 FBA 等规则，将卖家获取流量与使用其物流服务绑定，并用“不得外部更低价”的约束压制竞争，导致跨平台价格上升。作者还提到相关反垄断诉讼和临时禁令请求的意义，认为若法院支持，将迫使 Amazon 在审理前改变行为并加速案件推进。

  5. Anthropic Drops Flagship Safety Pledge
    comments

    报道称 Anthropic 将其 Responsible Scaling Policy（RSP）的核心承诺移除，不再坚持“在无法预先保证风险缓解到位时就不训练模型”的硬性规则，并强调这次调整是对竞争与现实环境的务实回应。新版政策提出更透明的安全披露、与竞争对手同等或更高的安全投入，并在被认为处于领先且灾难性风险显著时选择延迟开发，但整体约束力较旧版更弱。文章还提到公司在商业与技术上快速增长，内部与外部专家对“安全评估跟不上能力提升”的担忧依旧存在。

  6. Claude Code Remote Control
    comments

    该文档介绍 Claude Code 的 Remote Control：把本机正在运行的会话连接到手机、平板或浏览器，远程继续同一对话，且会话始终在本机运行，能使用本地文件、MCP 服务器和工具。它强调多端同步、断网或休眠后的自动重连，并说明与网页端 Claude Code 的区别在于执行环境在本机而非云端。文档给出使用前提（订阅与登录、工作区信任）、启动与连接方式（claude remote-control 或 /remote-control、会话 URL 与二维码），以及限制（单会话、终端需保持、长时间断网会结束）。

  7. Danish government agency to ditch Microsoft software (2025)
    comments

    丹麦政府数字事务部门的技术现代化机构计划以开源软件替换 Microsoft 产品，以降低对美国科技公司的依赖并推动数字主权。部长表示将于下月让过半员工从 Microsoft Office 迁移到 LibreOffice，若进展顺利将在秋季实现全员迁移，并力争年内完成。该决定也与 Windows 10 即将失去支持带来的维护成本、以及此前哥本哈根和奥胡斯两大市因成本与市场垄断担忧而弃用 Microsoft 的举动相呼应。同时欧洲多地正推进类似转向，例如德国石勒苏益格—荷尔斯泰因州改用 LibreOffice 与 Open-Xchange，并计划未来迁移到 Linux。

  8. Never buy a .online domain
    comments

    作者打破只买 .com 的习惯，因促销入手 .online 并快速上线，但站点突然被浏览器标记为不安全且域名进入注册局的 serverHold 状态，DNS 无法解析。为解除 Google Safe Browsing 的标记又必须先在 Google Search Console 验证域名，结果陷入“域名不解析就无法验证、无法验证就无法解封”的死循环，并因此批评 Radix 与 Google 的流程缺乏通知与容错。文中反思自己选了非常规 TLD、未尽早添加 Search Console、也未做可用性监控，并记录后来黑名单解除与站点恢复上线的更新。

  9. New accounts on HN more likely to use em-dashes
    comments

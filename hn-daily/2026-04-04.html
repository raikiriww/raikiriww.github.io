  1. We replaced RAG with a virtual filesystem for our AI documentation assistant
    comments

    这篇文章介绍了 Mintlify 如何把原本依赖检索增强生成的文档助手，改造成一个虚拟文件系统。作者认为，助手遇到跨页回答、精确语法或需要沿着文档结构主动探索时，单纯取回文本片段不够用，于是让页面和章节像文件与目录一样被浏览、搜索和读取。

    新方案名为 ChromaFs，基于已有的 Chroma 数据库把 ls、cd、find、cat 等命令映射为对文档索引的查询，从而让会话创建时间从约 46 秒降到约 100 毫秒。文章还强调了它带来的额外收益：复用现有基础设施、几乎没有边际算力成本，并能在构建文件树时顺带做权限过滤。

  2. OpenClaw privilege escalation vulnerability
    comments

    OpenClaw 在 2026.3.28 之前存在权限提升漏洞，问题出在 /pair approve 这一路径没有把调用者的权限范围传入核心审批检查。

    因此，只有配对权限、却没有管理员权限的用户，也可能批准那些要求更高权限范围的待处理设备请求，其中甚至包含管理员访问。

    漏洞与 extensions/device-pair/index.ts 和 src/infra/device-pairing.ts 中缺失的范围校验有关，实质上属于错误授权。

  3. iNaturalist
    comments

    iNaturalist 是一个面向自然观察者的社交平台，用户可以记录植物和动物的观察，并与朋友和研究者分享。平台强调把每条观察转化为可用于生物多样性研究的数据，还会同步到全球生物多样性信息设施等科研数据仓库，帮助科学家检索和使用。它还支持物种识别、建立生命清单、发起项目和举办生物普查，适合在手机等设备上持续记录自然发现。

  7. Embarrassingly simple self-distillation improves code generation
    comments

    这项研究提出了一个很简单的自蒸馏方法：先让模型用不同温度和截断参数自行采样生成解，再用这些样本做标准监督微调，而不依赖验证器、教师模型或强化学习。实验显示，Qwen3-30B-Instruct 在 LiveCodeBench v6 上的通过率从 42.4% 提升到 55.3%，且在更难题目上的收益更明显。作者还发现，这种方法对 Qwen 和 Llama 系列的 4B、8B、30B 模型都有效，并推断其作用机制与解码中的精度和探索冲突有关。

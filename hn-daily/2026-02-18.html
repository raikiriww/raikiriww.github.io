  1. Using go fix to modernize Go code
    comments

    文章介绍 Go 1.26 重写的 go fix 子命令，说明其通过一组分析器自动识别并应用更现代的语言与标准库特性，并建议在升级工具链后以干净的仓库状态运行，可用 -diff 预览改动。内容给出多类现代化示例，如用 strings.Cut、min/max、range over int、strings.Builder 等替代旧写法，并解释分析器可按需启用、不同平台可多次运行以覆盖更多构建配置。文中还强调 modernizer 设计会随版本演进（如 1.26 的 new(expr) 相关修复），并讨论多次运行产生协同改动、合并冲突与语义冲突的处理思路。

    comments

    AsteroidOS 2.0 发布，重点带来常亮显示、更多表款支持、新启动器样式、可定制快速设置、界面性能提升以及同步客户端增强。系统层面包含渲染优化、续航改进与稳定性修复，并补充了新增与实验级别表款的支持清单。同步端方面更新了 AsteroidOS Sync、Gadgetbridge、Amazfish 与 Telescope 等客户端功能，社区还贡献了表盘、应用与基础设施改进，并计划在未来完善路线图与发布节奏。

  3. BarraCUDA Open-source CUDA compiler targeting AMD GPUs
    comments

    BarraCUDA 是一个开源的 CUDA 编译器，面向 AMD GPU，当前可将 .cu 直接编译为 GFX11 机器码并生成可运行的 .hsaco。项目使用约 1.5 万行 C99 实现，完全不依赖 LLVM，包含从预处理、解析、语义分析到指令选择、寄存器分配与 ELF 输出的完整流水线。已支持大量 CUDA 语言与特性（如原子操作、warp 内置、共享内存等），同时列出若干尚未完成的语法与功能缺口，并给出优化与多架构扩展路线。

  4. Thousands of CEOs just admitted AI had no impact on employment or productivity
    comments

    文章以索洛生产率悖论为引子，指出新研究显示大量企业高管认为AI对就业与生产率尚无明显影响。调查涵盖多国企业，虽约三分之二高管称在用AI，但使用强度很低，近九成企业认为近三年对就业与生产率无影响，同时又预测未来三年仍有一定生产率与产出提升。作者还列举学术与机构研究给出相互矛盾的增益估计，并讨论竞争、使用方式与组织落地等因素可能决定AI是否出现“先抑后扬”的生产率曲线。

  6. Terminals should generate the 256-color palette
    comments

    作者主张终端应根据用户的 base16 主题自动生成 256 色调色板，以在简化配置的同时获得更丰富的色彩范围。文章指出默认 256 色与 base16 主题冲突、亮度与对比不一致，且 216 色立方体插值偏亮，影响可读性。解决方案是以 base16 的 8 个基础色、前景与背景色为基点，在 LAB 色彩空间中进行三线性插值生成 216 色立方体与灰度阶，从而保持一致亮度与可读性并兼顾兼容性。

    comments

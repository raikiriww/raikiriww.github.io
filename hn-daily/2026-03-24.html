  1. Autoresearch on an old research idea
    comments

    作者 Yogesh Kumar 实践了 Andrej Karpathy 的 Autoresearch 构想，利用 Claude Code 构建了一个自动化的“假设-训练-评估-迭代”闭环，用于优化其 eCLIP 研究项目。在受限的沙箱环境中，该代理通过 42 次实验将模型评价指标 Mean Rank 降低了 54%，其中最显著的提升源于代理自动发现并修复的代码 Bug。实验证明，LLM 代理在处理超参数微调等结构化任务时效率极高，但在应对复杂的架构创新和未知问题时仍面临局限性。

  2. FCC updates covered list to include foreign-made consumer routers
    comments

    美国联邦通信委员会（FCC）于2026年3月23日更新其受管制设备名单（Covered List），正式将外国制造的家用路由器纳入其中，并禁止批准此类产品的新型号。此项举措源于行政部门机构判定海外生产的消费级路由器对国家安全构成威胁，标志着美国进一步收紧了对关键通信硬件供应链的安全监管。

  5. Tell HN: Litellm 1.82.7 and 1.82.8 on PyPI are compromised
    comments

    热门 Python 库 litellm 的 1.82.7 和 1.82.8 版本在 PyPI 上遭遇供应链劫持，包含恶意 litellm_init.pth 文件。该文件利用 Python 特性在解释器启动时无需显式导入即可自动运行，大量窃取系统环境变量、SSH 密钥、云平台凭据及敏感配置文件。窃取的数据经加密后被发送至攻击者控制的伪造域名 litellm.cloud。受影响用户需立即核查本地环境并强制更换所有可能泄露的密钥与服务凭证。

  6. Malicious litellm_init.pth in litellm 1.82.8 PyPI package – credential stealer
    comments

    litellm 1.82.8 版本的 PyPI 软件包被发现包含恶意文件 litellm_init.pth，利用 Python 特性在解释器启动时自动运行，无需显式导入即可执行凭据窃取任务。该恶意脚本会搜集环境变量、SSH 私钥、主流云平台及 Kubernetes 凭据、Shell 历史和加密货币钱包等敏感数据，经 AES 与 RSA 加密后发送至攻击者控制的域名 models.litellm.cloud。这是一起严重的供应链攻击，受影响用户需立即卸载该版本并重置所有可能泄露的密钥与令牌。

  8. Apple Business
    comments

    苹果发布了定于2026年4月14日推出的 Apple Business 全合一平台，将整合并取代现有的 Business Essentials、Manager 和 Connect 服务，为全球企业提供设备管理与客户连接的统一方案。该平台内置移动设备管理（MDM）功能，支持通过管理式 Apple 账户实现工作与个人数据的加密隔离，并提供企业级自定义域名邮箱及协作工具。企业可统一管理在 Apple 地图、钱包和邮件中的品牌形象，利用位置洞察分析客户互动，并可于今夏起在美加地区投放地图本地广告。

  9. Wine 11 rewrites how Linux runs Windows games at kernel with massive speed gains
    comments

    Wine 11 引入了关键的 NTSYNC 核心驱动支持，通过在 Linux 内核层直接模拟 Windows 同步机制，大幅降低了多线程游戏的性能开销，在部分测试中实现了数倍的帧率提升。该版本完成了 WoW64 架构的重大重写，使得 64 位 Linux 系统无需安装 32 位库即可运行 32 位及 16 位 Windows 应用程序。此外，Wine 11 还显著优化了 Wayland 驱动的剪贴板和显示模式兼容性，并升级支持 Vulkan 1.4 规范，为 Linux 游戏生态提供了更高效、更稳定的底层支撑。

  10. Is anybody else bored of talking about AI?
    comments

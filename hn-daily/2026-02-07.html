  1. Start all of your commands with a comma (2009)
    comments

    作者长期在 ~/bin 中维护自定义脚本并加入 PATH，但担心与系统命令重名导致冲突，尤其在 Debian/Ubuntu 这类命令数量很大的环境中更明显。为避免冲突又保持易输入，他选择给自定义命令统一加上逗号前缀，指出逗号在文件名中是普通字符，各种工具与 shell 都能正常处理。借助 tab 补全，这样既能快速列出所有自定义命令，又能保证与系统命令完全区分，因此推荐该做法。

  3. Microsoft open-sources LiteBox, a security-focused library OS
    comments

    LiteBox 是微软开源的安全导向库操作系统，目标是通过大幅缩小与宿主的接口来降低攻击面，并支持内核态与用户态运行场景。它强调“North/South”分层接口与平台适配，提供类似 Rust 生态的接口风格，以便不同平台间灵活互通。示例用途包括在 Windows 运行未修改的 Linux 程序、在 Linux 上进行应用沙箱、在 SEV SNP 或 OP-TEE 等环境中运行程序，以及在 LVBS 上部署。项目仍处于快速演进阶段，接口可能变动，并以 MIT 许可证发布。

    How to talk to anyone and why you should

    文章以作者在火车与餐馆的两次陌生交流为引子，指出公共场合与陌生人交谈正在减少，背后有手机与社交媒体、远程工作、第三空间消失等结构性原因，也与“没人说我也不说”的社会规范强化有关。作者认为这会削弱人类的基本沟通能力，并引用心理学与神经科学相关观点说明年轻一代在对话与自我价值感上的困境。文章批评社交媒体上的“与陌生人对话挑战”过度表演化、工具化他人，反而加深疏离；同时给出低风险、小步尝试与尊重边界的实践建议，强调小型寒暄是维持社会连结与共同人性的微小但重要方式。

  2. 02
    Making Video Games in 2025 (without an engine)

    作者回顾自己长期不依赖大型商业引擎的原因，强调为小团队做定制化工具更轻量、可控且更能贴合游戏手感与工作流。文章具体介绍其技术栈与理由，包括以 C# 作为主语言、利用 dotnet 热重载提高迭代效率，并以 SDL3 作为跨平台底层抽象，自写一层渲染与输入工具。音频采用 FMOD，关卡编辑与工具界面依赖 Dear ImGui 或自建编辑器，并通过 C# 反射快速生成编辑面板；同时指出原生 AOT 编译和 SDL3 的主机平台支持降低了移植门槛。最后他说明已转向以 Linux 为主的开发环境，认为开源与跨平台生态更符合其长期可维护的开发理念。

  3. 03
    When does MCP make sense vs CLI?

    作者认为 MCP 并未带来现实收益，企业为新协议投入大量资源却仍需编写同样的工具说明，LLM 其实更擅长直接使用已有 CLI。CLI 对人和模型都友好，调试可复现、管道与组合能力强，适合处理大型数据流并复用现成工具链。MCP 在认证与运行时上引入额外复杂度，例如多次登录、权限粒度粗、服务进程不稳定，反而增加日常摩擦。作者承认在没有 CLI 的场景 MCP 仍有价值，但主张优先建设 API 与 CLI，让智能体自然接入。

    If AI writes code, should the session be part of the commit?

    该项目提供名为 git-memento 的 Git 扩展，用于在提交时记录 AI 编码会话，并把清理后的对话以 git notes 附加到提交上。它强调保持常规提交流程，同时支持多次 -m、amend、审核与诊断等命令，并可选择将完整会话与摘要分别存入不同的 notes 引用。工具支持可扩展的提供方配置与共享同步 notes，包含初始化、同步、携带 notes 以及 CI 审核等功能，并给出安装与发布说明。

    Everett shuts down Flock camera network after judge rules footage public record

    华盛顿州埃弗里特市在法院裁定Flock车牌识别摄像头影像属于公共记录后，已暂时关闭全市68个摄像头，市府称担忧公开申请可能被犯罪者或跟踪者利用并造成安全风险。该案源于居民提出公共记录申请，法院认定影像可被公众索取；市政府不同意裁定并表示若州议会立法豁免此类数据公开，将考虑重新启用。州议会正就限制披露与数据使用的法案进行辩论，支持者强调安全风险，反对者则指出申请流程冗长且难以用于实时追踪。

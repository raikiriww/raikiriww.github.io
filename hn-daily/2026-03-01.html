    MCP server that reduces Claude Code context consumption by 98%

    作者指出 Claude Code 的 MCP 工具调用会把原始输出塞满 200K 上下文，诸如 Playwright 快照、GitHub 工单列表和日志会迅速耗尽可用上下文。为此其构建 Context Mode 作为中间 MCP 服务器，把工具输出在沙箱中执行、只回传精简的 stdout，从而将 315 KB 输出压到 5.4 KB，宣称可降低约 98% 的上下文占用并显著延长可用会话时长。文章还描述了索引与搜索机制：按标题分块、保留代码块、基于 SQLite FTS5 与 BM25 排名并做词干化，支持从 URL 抓取后索引。

    Obsidian Sync now has a headless client

    Obsidian Sync 提供无界面 Headless Sync 客户端，用于在不打开桌面端的情况下同步仓库，适合 CI、代理和自动化流程，并支持一次性同步或持续监听更新。文档强调在启用前备份数据，且同一设备不要同时使用桌面端同步与 Headless Sync 以避免冲突。入门通过 npm 安装并使用 ob 命令登录、列出远端仓库、建立同步与运行连续同步，非交互场景可改用 OBSIDIAN_AUTH_TOKEN 环境变量。还说明了多项命令配置与状态查看，并介绍原生模块用于在 Windows 与 macOS 保留文件创建时间，Linux 不支持该能力。

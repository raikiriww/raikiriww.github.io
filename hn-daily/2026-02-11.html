Daily Hacker News for 2026-02-11

The 10 highest-rated articles on Hacker News on February 11, 2026 which have not appeared on any previous Hacker News Daily are listed below, with Chinese summaries for quick reading.

  1. Clean-room implementation of Half-Life 2 on the Quake 1 engine
    comments

    这是一个将《半衰期 2》移植到 Quake 引擎上的项目，当前版本并不能从头到尾完整游玩，但可以进行死亡竞赛等部分模式。它要求同时具备 hl2 和 hl2dm 目录，并建议直接使用 Steam 上的最新数据文件，旧版预发布数据则可打包为 pak0.pk3。项目还给出了运行与编译步骤：通过 fteqw.exe -halflife2 启动，并先构建 Nuclide，再在 Nuclide-SDK 中编译游戏逻辑与插件。

  2. The Feynman Lectures on Physics (1961-1964)
    comments

    卡尔理工学院与《费曼物理学讲义》网站推出了这套讲义的在线版本，让任何有网络和浏览器的人都能阅读费曼、莱顿和桑兹的经典内容。该版本专为不同尺寸的设备设计，文字、图表和公式都可以无损缩放，便于细看。页面还保留了章节中的讲座录音、照片和相关资源，并支持通过特殊链接恢复上次阅读位置。

  3. Ex-GitHub CEO launches a new developer platform for AI agents
    comments

    Thomas Dohmke 宣布成立新公司 Entire，并获得 6000 万美元种子轮融资，目标是为 AI 代理时代打造新的开发者平台。其首个产品是开源 CLI 工具 Checkpoints，核心作用是把代理会话中的提示、推理、文件变更、工具调用等上下文写入 Git，使这些原本会丢失的协作过程变得可追踪、可复用。文章强调，传统面向人类协作的软件交付流程已难以承载大量代理并行生成代码的现实，因此需要把上下文、约束和意图纳入版本化系统，作为未来开发流水线的基础。

  4. The Day the Telnet Died
    comments

    GreyNoise 观测到，2026年1月14日21:00 UTC 起，全球 telnet 流量在一小时内出现断崖式下跌，随后长期维持在比基线低约59%的水平，且多个自治系统和若干国家的数据直接归零。作者认为，这种突变更像是骨干网或转运网络层面的端口23过滤，而不是扫描行为自然衰减，因为云厂商影响较小、住宅和企业网络受影响更重。文章进一步把这一下降与六天后披露的 GNU Inetutils telnetd 高危认证绕过漏洞联系起来，推测可能存在提前通知后触发的基础设施层应对，但也明确表示这只能算推测，不能证明因果。

  5. Windows Notepad App Remote Code Execution Vulnerability
    comments

    该页面当前可提取到的主要内容并不是漏洞细节，而是 CVE 网站的通用说明、使用条款和版权提示。页面可见标题为 CVE Record: CVE-2026-20841，但摘录中没有给出该漏洞的成因、影响范围或利用方式。就现有可见文本来看，无法进一步还原具体技术信息，只能确认它指向一个与 Windows 记事本相关的远程代码执行漏洞条目。

  6. Chrome extensions spying on users' browsing data
    comments

    作者搭建了自动化扫描流程，在 Docker 中运行 Chrome，通过中间人代理观察扩展的外联请求，并用浏览器长短不同的网址长度来判断是否存在数据泄露。扫描结果显示，有 287 个 Chrome 扩展会泄露浏览记录，合计约 3740 万次安装，约占全球 Chrome 用户的 1%。文中还指出，这些泄露背后的主体涵盖 Similarweb、Curly Doggo、Offidocs、部分中文团体以及一些较小的数据中介，说明浏览器扩展被滥用来收集用户访问数据的问题仍然很严重。

  7. Fluorite – A console-grade game engine fully integrated with Flutter
    comments

    Fluorite 是一款与 Flutter 深度整合的游戏引擎，主打把游戏代码直接写在 Dart 中，从而降低开发复杂度，并继续利用 Flutter 的开发工具链。它采用以性能为导向的 ECS 架构，核心用 C++ 编写，目标是在低端或嵌入式硬件上也能保持较高表现。

    它还支持通过 FluoriteView 在同一场景中同时展示多个视图，并让游戏实体与界面组件共享状态。借助 Flutter/Dart 的热重载，开发者可以在很短时间内看到场景更新，提升迭代速度。

  8. GLM-5: From Vibe Coding to Agentic Engineering
    comments

    GLM-5 面向复杂系统工程和长周期代理任务，强调通过更大规模的预训练与后训练提升模型能力，并把参数规模、训练数据量和部署效率同时往前推进。相比 GLM-4.5，它扩展到 7440 亿参数、40 亿激活参数，并把预训练数据增加到 28.5 万亿 token，还引入 DeepSeek Sparse Attention，以降低部署成本并保留长上下文能力。

    该模型还配套了名为 slime 的异步强化学习基础设施，用来提升训练吞吐和后训练迭代效率。官方称 GLM-5 在多项基准上明显优于 GLM-4.7，尤其在推理、编程和代理任务上达到开源模型中的领先水平，并在内部评测和 Vending Bench 2 中展现出较强的长程规划与资源管理能力。

    GLM-5 已在 Hugging Face 和 ModelScope 开源，权重采用 MIT 许可，同时也可通过开发平台和 Z.ai 体验；它还支持与多种编码代理和本地推理框架配合使用。

  9. Claude Code is being dumbed down?
    comments

    文章批评 Claude Code 2.1.20 的改动：原本能直接看到文件读取和搜索模式的细节，现在默认只剩下“读取了 3 个文件”“搜索了 1 个模式”这类空泛汇总，几乎看不出工具到底做了什么。作者认为，这种做法既掩盖了代码库中的真实操作，也没有回应用户要求“恢复文件路径”或“提供开关”的诉求。相较于回退改动或增加简单配置，官方给出的“使用更详细模式”反而把大量调试输出一并端上来，和用户真正想要的信息并不匹配。文章还指出，多个版本不断削减详细模式的可读性，只是在用更复杂的方式绕回原本就该有的开关。

  10. Amazon Ring's lost dog ad sparks backlash amid fears of mass surveillance
    comments

    Ring 因在超级碗广告中宣传 Search Party 找狗功能而遭到集中批评，争议焦点是它把邻里摄像头网络包装成温情产品，却可能强化大范围监控。外界担心，结合 Ring 与 Flock Safety 的合作、以及近期推出的人脸识别功能，这类能力未来可能被扩展到搜索人类，进而服务于执法或更广泛的监视用途。Ring 则强调该功能只用于识别狗，不处理人类生物特征，且与 Familiar Faces 分开运行，但其默认开启、基于云端扫描的设计仍让隐私疑虑难以消散。

  1. Payphone Go
    comments

    该项目是一款以加州公用电话为目标的线下打卡游戏：用户注册获得 9 位玩家 ID，通过地图寻找电话亭，拿起话筒拨打指定的免费电话并输入 ID。系统依据来电号码识别电话亭并计分，首个打卡得 20 分、第二 10 分、第三 5 分，其后 1 分，排行榜实时展示玩家排名。作者解释数据来自向加州公共事业委员会提交的公开记录申请，但名单可能过时或遗漏，鼓励玩家反馈新地点以修正数据库。

  2. Hardening Firefox with Anthropic's Red Team
    comments

    文章介绍 Anthropic 与 Mozilla 合作强化 Firefox 安全：Claude Opus 4.6 在两周内发现 22 个漏洞，其中 14 个被评为高危，成果促成 Firefox 148.0 向数亿用户推送修复，并累计提交了 112 份报告。他们通过重现历史 CVE 与挖掘新漏洞验证模型能力，发现 AI 在漏洞发现上显著加速，但在利用上仍较弱。在数百次测试和约 4000 美元成本下仅两次成功构造粗糙利用，团队强调需借助任务验证器、最小复现和候选补丁等流程，提高报告可信度并加速修复。

  4. Helix: A post-modern text editor
    comments

    Helix 是一款强调多选区与多光标的终端文本编辑器，核心编辑基元是选择区，可并发操作代码。它集成 Tree-sitter 和语言服务器，提供更强的语法高亮、导航、自动补全与诊断等能力，并内置模糊查找、全局搜索与主题等现代功能。项目使用 Rust 构建，不依赖 Electron 或脚本语言，主打低配置与远程终端使用；同时说明与 Kakoune 和 Vim 的差异，并提到未来可能推出基于 WebGPU 的图形前端与插件计划。

  5. Plasma Bigscreen – 10-foot interface for KDE plasma
    comments

    Plasma Bigscreen 是面向电视、HTPC 与机顶盒的 Linux 开源大屏界面，可作为各类发行版的桌面环境部署。它提供适配遥控器或手柄的客厅设置与主页侧边栏，方便在沙发上管理显示、网络与应用切换。平台支持通过发行版包管理器或 Flathub 安装应用，示例包括 Steam、Kodi、Jellyfin 与经 VacuumTube 访问的 YouTube。项目强调开放栈与隐私保护，并由 KDE 社区志愿者协作开发，鼓励更多人参与贡献。

  6. LLMs work best when the user defines their acceptance criteria first
    comments

    作者以一次数据库基准测试说明，某个由 LLM 生成的 SQLite Rust 重写版在最基础的主键查询和插入上出现极端性能退化，原因并非语法错误而是规划器未把 INTEGER PRIMARY KEY 识别为 rowid、以及每次插入都触发昂贵的同步与重建流程，导致整体慢上数千到数万倍。他指出代码外观看似完整、模块齐全且测试通过，但在关键路径上存在设计与实现细节错误，并被一系列“安全但低效”的选择放大。作者由此强调 LLM 更擅长生成“看起来合理”的方案，若不在生成前明确验收标准与性能边界、并以基准与验证来约束，很容易产出满足描述却不满足实际需求的系统。

  7. UUID package coming to Go standard library
    comments

    作者提议在 Go 标准库中新增 crypto/uuid 包，用于生成和解析 UUID，范围涵盖版本 3、4、5。理由是第三方 github.com/google/uuid 在服务器与数据库类 Go 项目中几乎是必备依赖，接口多年稳定，且 UUID 本身是标准。补充指出，相较于 C#、Java、JavaScript、Python、Ruby 等语言，Go 在标准库缺乏 UUID 支持显得较为例外。

  8. Uploading Pirated Books via BitTorrent Qualifies as Fair Use, Meta Argues
    comments

    在作者起诉Meta的集体诉讼中，法院曾认可其用盗版书训练Llama的合理使用，但仍留下通过BitTorrent下载并上传的直接侵权争议。Meta最新向法院提交补充答复，主张在下载过程中自动向他人上传是技术机制所致且为获取数据所必需，因此应同样构成合理使用；作者方则指其在期限后新增抗辩。双方就这一抗辩是否早已提出及其合法性产生新的程序争议，案件后续将由法官决定是否允许继续以该理由抗辩。

  9. Ki Editor - an editor that operates on the AST
    comments

    Ki Editor 是一款多光标的结构化编辑器，核心是围绕语法树节点进行操作，从而直接操控语法结构以贴近编码意图。它强调多光标并行处理语法节点，提升批量编辑与重构效率。编辑方式上通过统一的选择模式覆盖词、行、语法节点等不同粒度，带来一致而灵活的操作体验。
